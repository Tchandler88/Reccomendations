🔐 Roles, Permissions & ACL Architecture

The current permission model relies heavily on tag-based Closed User Groups (wcm-tags:security, ~200 tags) which is a legacy AEM pattern incompatible with AEM Cloud Service's group-based ACL model. A full ACL rebuild is required before the tag migration can be completed. This panel documents the proposed role model and migration prerequisites.

🔴 MIGRATION BLOCKER — DQ-07: The wcm-tags:security namespace must remain intact until a replacement group-based ACL model is fully designed, built in Stage, and validated. This is the final migration phase — schedule it LAST, after all other namespace retirements are complete.

DQ-06 — wcm-tags:advisor Personalization Risk

⚠ HIGH RISK — Advisor Portal Personalization: The wcm-tags:advisor territory tags (~55) may be actively driving personalized content delivery in the advisor portal via Adobe Target audience rules or AEM Context Hub segment definitions. If these tags are retired before the personalization rules are rewritten to read from the new wcmAdvisorTerritory schema field, advisor portal content targeting will silently fail. Required action: WCM developer must audit all Adobe Target audience rules and Context Hub configurations before retirement begins. The DAM migration may proceed in parallel — assets can carry both the old tag and new schema field during transition.
